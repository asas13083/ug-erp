# دليل رفع نظام UG ERP على سيرفر حقيقي (Docker)

آخر تحديث: 11 يوليو 2026

---

## ليه Docker بس، من غير PM2 منفصل؟

PM2 شغلته الأساسية إنه "يراقب" برنامج الـNode ويرجّعه يشتغل لو وقع أو السيرفر اتعمله إعادة تشغيل. الـDocker بيعمل بالظبط نفس الوظيفة دي عن طريق سطر `restart: unless-stopped` الموجود في `docker-compose.yml` — يعني أي حاوية (container) تقع أو السيرفر يريستارت، Docker بيشغّلها تاني لوحده تلقائياً. فمفيش داعي لأداتين بيعملوا نفس الحاجة بالظبط — ده بيبسّط الموضوع بدل ما يعقّده.

---

## الملفات اللي اتضافت

```
line-media-erp/
├── docker-compose.yml          ← الملف الرئيسي، بيشغّل كل حاجة بأمر واحد
├── .env.example                ← مثال لباسورد قاعدة البيانات (لـ Docker بس)
├── nginx/
│   └── default.conf            ← إعداد الاستقبال والتوجيه
├── backend/
│   ├── Dockerfile
│   └── .dockerignore
└── frontend/
    ├── Dockerfile
    └── .dockerignore
```

## الخطوات (على السيرفر، مرة واحدة أول ما تجهّزه)

### 1. ثبّت Docker على السيرفر
لو السيرفر Ubuntu (الأشهر والأرخص على DigitalOcean مثلاً)، شغّل السطر ده:
```bash
curl -fsSL https://get.docker.com | sh
```

### 2. ارفع ملفات المشروع كلها على السيرفر
عن طريق `git clone` لو المشروع على GitHub، أو `scp`/`rsync` لو بترفع الملفات يدوياً من جهازك.

### 3. جهّز ملفات الإعدادات (مرة واحدة بس)

انسخ `.env.example` (اللي في جذر المشروع) لملف اسمه `.env` في نفس المكان، واملا باسورد قوي لقاعدة البيانات:
```bash
cp .env.example .env
nano .env
```

وتأكد إن `backend/.env` موجود ومظبوط (نفس الملف اللي عندك على جهازك دلوقتي — انسخه للسيرفر، بس **مش لازم يكون فيه DATABASE_URL صحيح، لأن docker-compose بيحدده تلقائياً**).

### 4. شغّل كل حاجة بأمر واحد
```bash
docker compose up -d --build
```
`-d` معناها يشتغل في الخلفية، و`--build` معناها يبني الصور الأول مرة (أو بعد أي تحديث كود).

### 5. طبّق الجداول على قاعدة البيانات (أول مرة بس)
لو الـmigrate deploy مجاش تلقائي (بيتحصل في الـDockerfile نفسه عادةً)، تقدر تشغّله يدوي:
```bash
docker compose exec backend npx prisma migrate deploy
```

### 6. (اختياري) زرع بيانات أولية — مستخدم أدمن مثلاً
```bash
docker compose exec backend npm run seed
```

## أوامر تشغيل يومية بعد كده

| العملية | الأمر |
|---|---|
| تشغيل كل حاجة | `docker compose up -d` |
| إيقاف كل حاجة | `docker compose down` |
| إعادة التشغيل بعد تحديث كود جديد | `docker compose up -d --build` |
| متابعة السجلات (Logs) لحظياً | `docker compose logs -f backend` |
| الدخول جوه حاوية الباك إند (لأي أمر يدوي) | `docker compose exec backend sh` |
| تشغيل migration جديدة بعد تحديث | `docker compose exec backend npx prisma migrate deploy` |

## SSL (https) — خطوة لاحقة موصى بيها

بعد ما يكون عندك دومين (اسم موقع) موجّه لعنوان IP بتاع السيرفر، أسهل طريقة تضيف https مجاناً هي **Certbot**:
```bash
sudo apt install certbot python3-certbot-nginx -y
```
وبعدين هنحتاج نعدّل `nginx/default.conf` شوية ونضيف حاوية Certbot — ده الخطوة الجاية بعد ما يكون عندك دومين فعلي جاهز، قولّي وقتها ونظبطها مع بعض.

## ملحوظة عن النسخ الاحتياطي جوه Docker

زرار "استرجاع" و"تشغيل نسخة احتياطية الآن" جوه البرنامج هيشتغلوا عادي زي ما هما — الحاوية بتاعة الباك إند فيها `pg_dump` و`psql` مثبّتين جواها. الملفات نفسها بتتخزن في `backend/backups` على السيرفر نفسه (مش جوه الحاوية)، فلو الحاوية اتبنت تاني، النسخ الاحتياطية متضيعش.
